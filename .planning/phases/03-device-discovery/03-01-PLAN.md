# Plan 03-01: WebRTC Signaling Server Implementation

<context>
This plan sets up `socket.io` on the established Express server to act as a WebRTC signaling server. This is foundational for devices communicating dynamically and grouping together using their IP connection data before establishing a direct P2P connection later.
</context>

<requirements>
- [ ] Establish socket.io handling in `server.js` matching standard WebSocket patterns via an `http` package wrapped instance.
- [ ] Group connecting devices into the same "Room" utilizing their `socket.handshake.address` strings (IP values).
- [ ] Broadcaast `device_joined` events when new sockets attach to rooms.
- [ ] Maintain an active state on the server representing all currently loaded sockets/devices within their networks.
</requirements>

<must_haves>
- Devices on exactly the same network segment must natively join the same logical `socket.io` room securely.
- WebSockets must serve via the same 3000 port utilized for the UI without any additional port allowances.
</must_haves>

<tasks>
<task>
Run `npm install socket.io` adding it to package.json.
</task>
<task>
Require `http` and `socket.io` in `server.js`.
1. `const http = require('http');`
2. `const { Server } = require('socket.io');`
3. Convert `app.listen()` initialization: Instantiate `const server = http.createServer(app);`, then `const io = new Server(server);`, and change listener to `server.listen(port, host, () => {...})`.
</task>
<task>
Implement `io.on('connection', (socket) => { ... })` handler.
1. Read `const ip = socket.handshake.address;`
2. Call `socket.join(ip);` placing them into a networked room.
3. Emit within that room `socket.to(ip).emit('device_joined', { id: socket.id })`.
4. Handle `disconnect` and emit `device_left`, strictly wiping the socket ID and any associated session fragments from server memory. Ensure absolutely NO temporary files, images, or caches are ever written to the server's disk during connection or upon disconnect.
</task>
</tasks>

<verification>
1. Launch `node server.js` ensuring server starts reliably using `http.createServer` logic.
2. Confirm the server listens effectively on `0.0.0.0:3000` exactly as before.
</verification>
