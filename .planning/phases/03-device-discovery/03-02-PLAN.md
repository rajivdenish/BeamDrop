# Plan 03-02: QR Pairing Flow & Device List UI

<context>
Using the established Socket.io server from 03-01, we want to allow users to generate a dynamic QR pairing ticket on the Web UI that a secondary device effectively scans to "Pair", registering itself inside the local discovery module visually as a connected instance UI element.
</context>

<requirements>
- [ ] Implement `socket.io-client` on `public/index.html`.
- [ ] Add a visual button to generate a pairing QR code in the browser containing a random `token` and the host `ip:port`.
- [ ] Handle URL parsing when rendering `index.html` ŌĆö if `?pair=token` exists, trigger the socket pairing initialization automatically.
- [ ] Update UI with a visual list of discovered devices connected to the same IP.
</requirements>

<must_haves>
- Uses `qrcode.js` library or an alternative CDN hosted QR code script for client-side rendering.
- Visual updates displaying "Connected Device: [ID]" dynamically upon pairing MUST match the premium design (using CSS cards, micro-animations, and hover states).
- The backend signaling server tracks paired tokens in-memory ensuring a token is valid, executing the discovery `socket.join()` dynamically.
</must_haves>

<tasks>
<task>
Update `server.js` to handle signaling and pairing logic explicitly.
1. Add an in-memory `pairingTokens = {}` object tracking who requested the pair.
2. In `io.on('connection')`, allow sockets to emit a `generate_pairing` event that returns a unique securely randomized string, saving it to `pairingTokens` mapping to that socket's IP group.
</task>
<task>
Update `public/index.html` fetching the Socket.io client script: `<script src="/socket.io/socket.io.js"></script>`. Include `<script src="https://cdn.jsdelivr.net/npm/qrcode@1.5.3/build/qrcode.min.js"></script>` or similar for UI QR Code rendering.
</task>
<task>
Add UI elements in HTML: A button to "Pair new device", a canvas for QR Code `id="qrCanvas"`, and an empty `ul` block `id="deviceList"` displaying currently tracked devices.
</task>
<task>
In `index.html`, initialize `const socket = io();`.
1. Bind the "Pair" button to emit `generate_pairing`.
2. Upon receiving the token, dynamically render the URL `http://<lanIp>:<port>/?pair=token` natively onto the `qrCanvas`.
3. Check `window.location.search` for `?pair=` parameter heavily caching pairing states into `localStorage.setItem('deviceId', socket.id)` and verifying.
</task>
</tasks>

<verification>
1. Launch app on PC, click "Pair Device". A QR Code will appear.
2. Use a mobile simulator or another browser to hit `http://192.168.100.100:3000/?pair=<TOKEN>`.
3. Ensure both the host and the simulated client log the successfully discovered device connection in the `console`.
</verification>
