# Plan 06-03: End-to-End Transfer Smoke Test (COMPAT-03)

## Context
Execute a complete walkthrough simulating the exact flow a true user experiences passing files and text simultaneously between Apple iOS Safari and Windows 11 Chrome.

## Implementation Details
1. **Initial Spin-Up**: Ensure `.server.js` starts from scratch without crashing under HTTP/S configuration and outputs correct IPv4 LAN.
2. **Auth & Pair**: Verify manual login persistence works, and pairing the device on iOS over HTTPS (ignoring self-signed warnings securely).
3. **Dual Link**: Ensure both device lists populate without duplicate socket representations upon hard refreshing either tab. The Discovery system must clean up old WebRTC states proactively.
4. **Transfer matrix**:
   - PC -> Text -> Phone (Receive and render correctly)
   - Phone -> Text -> PC (Receive to Inbox instantly)
   - PC -> Big File -> Phone (Check memory buffer limits over 150MB logic loop)
   - PC -> Ctrl+V -> Phone
   - Phone -> Disconnect WiFi -> Wait 15s -> Check Dead peer handling -> Reconnect WiFi -> Auto-re-pair the WebRTC connection successfully.

## Success Criteria
- [ ] No unhandled Promise exceptions occur during heavy DataChannel payload sending.
- [ ] iOS Safari gracefully suspends and reawakens its persistent connection safely using background timeout recovery handlers seamlessly.
