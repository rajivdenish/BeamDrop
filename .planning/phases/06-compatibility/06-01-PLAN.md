# Plan 06-01: iOS Safari Compatibility (COMPAT-01)

## Context
iOS Safari natively behaves differently than Chrome when dealing with Blob URLs. Safari often attempts to render known MIME types inline within the browser tab rather than prompting a "Download / Save As" mechanism. Additionally, backgrounding tabs on iOS aggressively suspends WebRTC channels and JavaScript event loops. This plan aims to stabilize these behaviors.

## Implementation Details
1. **File Downloading Adjustments**:
   - The current `saveReceivedFile` uses `window.showSaveFilePicker` which is desktop-only. For mobile, the `renderPreview` provides an `<a>` tag with a `download` attribute.
   - We must ensure the `href` with `URL.createObjectURL(blob)` effectively triggers a true download payload. Safari handles the `download` attribute well in modern iterations, but we must verify the behavior and provide clear instructions.
   - For `Blob` instances, when a user clicks download, if iOS opens it inline, they'll need to hit "Share -> Save to Files". Consider overriding `mime` to `application/octet-stream` on the `<a>` tag explicitly if needed to force a download prompt, but `text/plain`, images, and PDFs generally trigger native quick-look.
2. **Tab Suspension and Recovery**:
   - iOS suspends JavaScript execution after a few seconds of backgrounding.
   - The 5-second WebRTC ping/pong mechanism will naturally terminate the connection if no response is received in 15 seconds.
   - If a peer disconnects, we need a method to rapidly re-establish on foregrounding using the `visibilitychange` event listener.
   - We will implement a `document.addEventListener('visibilitychange', ...)` handler that automatically re-emits `request_sync` or re-establishes signaling if the socket was dormant.

## Success Criteria
- [ ] iOS Safari successfully views Image/PDF previews and can download the blobs using the "Download Blob" button without crashing.
- [ ] When an iOS device minimizes the tab for >20 seconds, the PC marks it disconnected. When the iOS device foregrounds again, it automatically re-announces its presence to the signaling server and repairs its state on the PC.
