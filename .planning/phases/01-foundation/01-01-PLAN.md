# Plan 01-01: Node.js/Express Server Scaffold

<context>
This plan sets up the foundational Express server for the Web Data Transfer project. It initializes the Node.js project, implements the Express app bound to `0.0.0.0` for LAN access, and serves static files from a `public` directory.
</context>

<requirements>
- [ ] Initialize package.json and install `express`.
- [ ] Create `server.js` binding an Express app to `0.0.0.0` and a default port (e.g., 3000).
- [ ] Create a barebones `public/index.html` file to verify static serving.
- [ ] Setup initial unhandled promise rejection and basic error logging.
</requirements>

<must_haves>
- Server must bind to `0.0.0.0`, not `127.0.0.1`, allowing incoming connections from the local network.
- `public/index.html` reachable from `http://localhost:3000` locally.
</must_haves>

<tasks>
<task>
Run `npm init -y` and `npm install express` to set up the foundation of the server.
</task>
<task>
Create `server.js` requiring Express and starting a server listening on `process.env.PORT || 3000` and host `0.0.0.0`. Log a success message to the console on startup.
</task>
<task>
Create `public/index.html` with a basic HTML skeleton including a title "Web Data Transfer" to verify functionality.
</task>
<task>
Add `app.use(express.static('public'))` to `server.js` so `index.html` can be rendered.
</task>
<task>
Add a basic starting script in `package.json`: `"start": "node server.js"`.
</task>
</tasks>

<verification>
1. Run `npm start`.
2. Ensure the terminal logs "Server listening on port 3000".
3. Check `http://127.0.0.1:3000` or `http://localhost:3000` and see "Web Data Transfer" title rendered correctly.
</verification>
