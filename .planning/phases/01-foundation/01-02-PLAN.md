# Plan 01-02: LAN IP Detection Heuristics

<context>
Building on the Express scaffold, the server needs to determine the correct Wi-Fi/LAN IP address. Real networks have multiple virtual NICs (WSL, Docker, VPN), and identifying the correct adapter is crucial so mobile devices can reach the server.
</context>

<requirements>
- [ ] Implement LAN IP detection function in Node using `os.networkInterfaces()`.
- [ ] Filter out virtual network interfaces (e.g. `vEthernet`, `WSL`, `Docker`).
- [ ] Ignore internal loopback addresses (`127.0.0.1`).
- [ ] Select the first valid IPv4 address mapped to the found interface.
- [ ] Provide a `--host` argument override that bypasses heuristics if specified.
</requirements>

<must_haves>
- The heuristically found LAN IP address must be logged on the console cleanly on server start.
- If `--host <IP>` is provided via `process.argv`, it overrules any logic and immediately uses `<IP>`.
</must_haves>

<tasks>
<task>
Require `os` built-in module inside `server.js`.
</task>
<task>
Parse `process.argv` to check for `--host` arguments. Example `--host 192.168.1.100`. Set `lanIp` to this value if provided.
</task>
<task>
If no `--host` arguments are given, call `os.networkInterfaces()`, filter out keys representing virtual nets (case-insensitive checks on names like `vEthernet`, `WSL`, `Docker`), and retrieve the IPv4 address from the non-internal networks. If multiple legitimate external interfaces exist, choose the first. Default to `localhost` if none found.
</task>
<task>
Log the final `lanIp` to the console ("Server is reachable on LAN at `http://<lanIp>:3000`").
</task>
</tasks>

<verification>
1. Start `node server.js` and verify it logs `192.xxx` or equivalent valid local IP.
2. Start `node server.js --host 10.0.0.2` and verify the log now specifically reflects `10.0.0.2`.
</verification>
