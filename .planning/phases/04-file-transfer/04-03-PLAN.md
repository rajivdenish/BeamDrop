# Plan 04-03: WebRTC Chunking & Real-Time Progress UI

<context>
Because WebRTC DataChannels often crash or fail to flush properly if you attempt to push a massive 5GB file blindly into `peer.send()`, we must explicitly chunk binary payloads. We'll use JavaScript `FileReader` to stream small `ArrayBuffer` slices natively from disk to WebRTC buffers, displaying an aesthetic realtime calculation visually per chunk transmitted.
</context>

<requirements>
- [ ] Determine a safe chunk size (e.g. 16KB - 64KB) for WebRTC messages to guarantee delivery reliability across differing routers.
- [ ] Before a file transmits, send a stringified JSON schema: `{ type: 'file-metadata', name: 'video.mp4', size: 104857600, mime: 'video/mp4' }`.
- [ ] Transmit `Uint8Array` chunks iteratively via `peer.send(chunk)`.
- [ ] The receiver should cache an array of `chunks = []` until the accumulated byte length intrinsically matches the incoming metadata.
- [ ] Render a sleek CSS Progress Bar visually mapping `chunksSent / totalSize` via width percentages.
</requirements>

<must_haves>
- The progress bar MUST be fluid (`transition: width 0.1s linear`), styled cleanly matching `var(--primary-color)`.
- When transferring multiple files consecutively, wait for the receiver's acknowledgment before slamming the port (optional buffer management). Alternatively, serialize file queues natively.
- Only construct the final visual DOM `Blob` and URL dynamically when 100% of the byte chunks resolve successfully.
</must_haves>

<tasks>
<task>
In `public/styles.css`, construct a `.progress-bar-container` (bg `#E5E7EB`, height `8px`, `border-radius: 4px`) and an inner `.progress-fill` (bg `var(--primary-color)`, `width: 0%`, `transition`).
</task>
<task>
In `public/index.html`, write `sendNextChunk(file, offset)` method. It loads `file.slice(offset, offset + 64000)` using `FileReader.readAsArrayBuffer()`. On read complete, call `peer.send(e.target.result)` and advance `offset`.
</task>
<task>
Add logic binding `updateProgress(bytesLoaded, totalBytes)` injecting style updates to the corresponding HTML row visually.
</task>
<task>
On receiver side, when `typeof data !== 'string'` arrives from `peer.on('data')`, push it into an active `incomingChunks` buffer. Add data length to `bytesReceived`. Update its own visual progress UI identically. If `bytesReceived === activeMeta.size`, execute `new Blob(incomingChunks)`, build the rich UI preview, and reset buffers for consecutive queues.
</task>
</tasks>
