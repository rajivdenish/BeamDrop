# Plan 04-02: File Selection & Rich Preview UX

<context>
With WebRTC tunnels functional, we need a premium mechanism for users to declare *what* they are sending. The requirements specify a button (File Picker) and physical drag-and-drop mechanics onto a target device card. Furthermore, any received arrays mapped back to the UI must instantiate visually rich media wrappers (Preview & Play).
</context>

<requirements>
- [ ] Create a hidden `<input type="file" multiple>` attached to a customized "Send Files" mechanism that becomes visible when a P2P Link selects its target.
- [ ] Implement Drag-and-Drop: Highlight the specific `device-card` or overall `dashboard-container` when files are hovered over it. Extract the `dataTransfer.files` efficiently.
- [ ] For every file selected (dragged or picked), build a dynamic UI row predicting the queue transfer.
- [ ] When a file is received via WebRTC, automatically parse its `MIME` type to inject a `Blob` URL natively into an interactive view block (`<img>` tag for photos, `<video controls>` for MP4/WebM, `<audio controls>` for MP3, else a generic File Download anchor `<a>`).
</requirements>

<must_haves>
- Adhere strictly to the Premium Interface theme. Modal popups or expanding file queues should mimic `var(--card-bg)` shadow depth.
- Previews must execute entirely off RAM. 
</must_haves>

<tasks>
<task>
In `public/styles.css`, construct new classes mapping to an aesthetic file queue list (`.file-item`, `.file-thumbnail`, `.drag-active` glowing border class).
</task>
<task>
In `public/index.html`, append a modal or an expanding div bound uniquely to the active `peer`. Implement `input.click()` delegating file selection to the OS picker without showing the ugly default button.
</task>
<task>
Attach native JS EventListeners: `dragover`, `dragenter`, `dragleave`, `drop` explicitly preventing defaults, extracting `e.dataTransfer.files`, passing them directly mapping to the queue function.
</task>
<task>
Create a DOM constructor `createFilePreview(fileBlob, metadata)` triggering when a full WebRTC transmission completes. Use `URL.createObjectURL(fileBlob)` conditionally injecting `<img/>` or `<video/>` nodes.
</task>
</tasks>

3. Validate OS native file picker functionality.
