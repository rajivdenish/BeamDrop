# Plan 04-01: WebRTC P2P Tunnels Setup

<context>
Building upon Phase 3's `socket.io` discovery list, we need to utilize those sockets as pure signaling channels to exchange WebRTC ICE candidates and Session Descriptions (SDP). This establishes a direct browser-to-browser P2P tunnel exactly matching the zero-server-storage privacy goal. 
</context>

<requirements>
- [ ] Incorporate `simple-peer` via CDN inside `public/index.html` as the standard, robust abstraction for native WebRTC.
- [ ] Upon clicking a discovered "Device Card" in the UI, the initiator generates a WebRTC "Offer", sending it over `socket.io` to the specific target device's socket ID.
- [ ] Add `signal` pass-through routes mapped to specific `to` and `from` IDs strictly on the `server.js` backend, never logging or storing the signal payloads.
- [ ] The receiver catches the `Offer`, generates an `Answer`, and routes it back over `socket.io` to seal the P2P connection tunnel.
</requirements>

<must_haves>
- The backend `server.js` functions merely as an ignorant relay tube (`socket.to(target).emit('signal', payload)`).
- When a P2P tunnel connects, change the device card status visually from "● Connected" (Socket) to a glowing green "● P2P Linked" using premium aesthetics.
</must_haves>

<tasks>
<task>
In `server.js`, add `socket.on('signal', (data) => { socket.to(data.to).emit('signal', { from: socket.id, signal: data.signal }); });`.
</task>
<task>
In `public/index.html`, add `<script src="https://cdnjs.cloudflare.com/ajax/libs/simple-peer/9.11.1/simplepeer.min.js"></script>`.
</task>
<task>
In `index.html`, bind click events to newly rendered Device Cards. Clicking a card initiates `const peer = new SimplePeer({ initiator: true, trickle: false })`.
1. Bind `peer.on('signal', data => socket.emit('signal', { to: targetId, signal: data }))`.
2. Map `socket.on('signal', data => { ... })`: if not initiator, create a new `SimplePeer({ initiator: false })`, then call `peer.signal(data.signal)`. Respond by emitting its own signal back to the sender.
</task>
<task>
Attach a `peer.on('connect')` event handler that visually updates the UI for both peers indicating that the secure data tunnel is successfully open and ready for file chunks payload.
</task>
</tasks>
