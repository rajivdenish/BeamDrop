# Plan 02-02: Login, Session Persistence, Logout & Admin Interface

<context>
Building on the registration capabilities of `02-01`, we need robust login mechanisms utilizing secure cookies to authorize incoming requests. We install `cookie-parser` and setup `HttpOnly` sessions tied to locally managed tokens matching the `db.json` users without expiring upon single browser tabs closing. A frontend interface logic `login.html` and `register.html` is constructed alongside the admin approval layer.
</context>

<requirements>
- [ ] Add `cookie-parser` middleware and enforce session tokens mapping.
- [ ] Implement `POST /api/auth/login` handling email/password and returning a session cookie.
- [ ] Implement `POST /api/auth/logout` clearing the session cookie.
- [ ] Create `/api/admin/approve/:userId` authorizing pending registrations.
- [ ] Inject session-protection middleware across all future routes.
- [ ] Front-end `login.html` and `register.html` static pages handling fetch calls securely.
</requirements>

<must_haves>
- Uses `cookie-parser` inside Express avoiding manually parsing Set-Cookie strings.
- Sets standard `HttpOnly; SameSite=Strict` cookie logic preserving state across tabs.
- The `requireAuth` server middleware prevents access to the core app (`/`) for unregistered/unauthenticated users.
- Standard Vanilla JS and HTML for login elements (no UI frameworks, no build steps).
</must_haves>

<tasks>
<task>
In `server.js`, `app.use(require('cookie-parser')())` to easily fetch Session IDs. Create a simple `requireAuth` Express middleware inspecting `req.cookies.sessionId`, comparing it against the local JSON DB `sessions` mappings. If invalid, throw `401 Unauthorized` or redirect to `/login.html`.
</task>
<task>
Implement `POST /api/auth/login` in `server.js`.
1. Parse `{email, password}` from JSON.
2. Locate user and verify `hashPassword` hash.
3. Determine if their `status` equals `approved`. Returns `403` if currently `pending`.
4. Generate `sessionId` via `crypto.randomBytes(32).toString('hex')`.
5. Insert into db `sessions: { [sessionId]: userId }` + File Write.
6. `res.cookie('sessionId', sessionId, { httpOnly: true, sameSite: 'strict', maxAge: 86400000 })`. Resolves `200 OK`.
</task>
<task>
Implement `POST /api/auth/logout` eliminating the cookie via `res.clearCookie('sessionId')` terminating session tracking. Include the logic removing it from the `db.json` completely.
</task>
<task>
Implement `POST /api/admin/approve` using the `requireAuth` middleware to verify the invoking user's role is `admin`. Map over the user DB and set the requested ID's status to `approved`.
</task>
<task>
Generate `public/login.html` and `public/register.html` files with simple vanilla HTML forms mapping payload fetches to the API endpoint and resolving to `index.html` locally.
</task>
<task>
Update `app.use(express.static('public'))` routing in the server architecture to not freely serve the index without authentication (e.g. selectively serving the login/register first, rendering HTML only upon verified login auth token checks).
</task>
</tasks>

<verification>
