# Plan 02-01: User Model, Registration & Invite Validation

<context>
This plan sets up a simplistic flat-file database using Node's native `fs` to persist users and invite logs, fulfilling the zero-dependency requirements. It establishes the password hashing methodology using `crypto` and the fundamental user registration API ensuring only invited users or admins can access features.
</context>

<requirements>
- [ ] Implement `auth.js` helper exporting saving/loading JSON logic for `db.json` (users, sessions, invites).
- [ ] Implement a `POST /api/auth/register` route capturing `{email, password, inviteToken}`.
- [ ] Create the first registered user automatically as an Admin.
- [ ] Hash passwords using `crypto.scryptSync`.
- [ ] Restrict registration: Users must provide a valid `inviteToken` unless they are the first user (admin). If invalid, return a `403 Forbidden` response.
- [ ] Generate invite tokens and validate their usage upon registration.
</requirements>

<must_haves>
- Uses local `db.json` files for offline standalone operation without third-party DB setups.
- First user registered invariably attains the role of `admin`.
- Native Node.js hashing (using `crypto`) instead of Native Addons (e.g., `bcrypt`).
- API returns HTTP 403 on invalid invite codes during registration.
</must_haves>

<tasks>
<task>
Create `auth.js`. Write functions to read/write a `db.json` object structured as `{ users: [], invites: [], sessions: {} }`. Use `fs.existsSync` to initialize an empty structure if it doesn't exist yet. Use `crypto.randomBytes(16).toString('hex')` for tokens.
</task>
<task>
In `auth.js`, export helper functions for `hashPassword(plaintxt)` using Node's `crypto.scryptSync` and returning a saled hash string, and `verifyPassword(plain, hashed)` to secure user passwords.
</task>
<task>
In `server.js`, import `express.json()` and `auth.js` to begin constructing the auth API endpoints.
</task>
<task>
Implement `POST /api/auth/register` in `server.js`.
1. Parse the JSON body.
2. Load the database.
3. If `users` array is empty, grant `role=admin`, `status=approved`, and bypass invite token check.
4. If `users` has length > 0, check the `inviteToken` against the `invites` object to ensure it is valid, not used, and not expired. If invalid throw `403`. 
5. Hash the password.
6. Push new user to the DB and update the `inviteToken` status to `used`. Save the file. Returning a `201 Created`.
</task>
<task>
Implement a simple `POST /api/admin/invite` route taking an admin validation check (bypass auth token check temporarily here for testing, or assume hardcoded first) generating an invite string, pushing to the DB `invites` array, and returning it.
</task>
</tasks>

<verification>
1. Start the server. No `db.json` will exist initially.
2. POST to `/api/auth/register` without an invite code. Assuming users array is empty, it should resolve to 201 Created and save the admin to `db.json`.
3. Try registering a second account without an invite code. Ensure it returns 403 Forbidden.
</verification>
