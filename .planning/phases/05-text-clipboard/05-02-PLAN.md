# Plan 05-02: PC-Side Ctrl+V Paste-to-Send Shortcut (TEXT-02)

## Context
Provide the PC-hosted browser instance an absolute global listener to instantly seize user clipboard text content anytime they type `Ctrl+V` (or `Cmd+V`) natively onto the site and securely forward it strictly to one primary paired device.

## Implementation Details

### Step 1: Global Clipboard Listener (`public/index.html`)
- Attach a `window.addEventListener('paste', ...)` directly to the application.
- When the event fires, ensure no input boxes are actively selected (to avoid intercepting standard input operations during file renaming).
- Extract the text using `e.clipboardData.getData('text')`.

### Step 2: Connection Target Validation
- Filter all currently `connected` P2P `SimplePeer` objects inside the `peers` Map.
- If none exist, abort.
- If one exists and is fully linked, automatically execute the `peer.send()` logic to broadcast the clipboard string instantly.
- Show a local confirmation using `renderPreview(..., "Clipboard Sent Natively")`.

## Success Criteria
- [ ] A PC user clicks on their app container anywhere (focusing on `body` or `div`), taps `Ctrl+V`, and the clipboard block seamlessly shoots into the connected WebRTC pipe to the mobile phone.
